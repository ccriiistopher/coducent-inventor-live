Fancy RecyclerView 2

It is supported for:KodularAppInventorNiotron
It is part of Fancy RecyclerView
To get this extension, please go to: Fancy RecyclerView

Posted on December 6, 2021 4:46 PM

Based on the Android RecyclerView widget. This extension adds many functionalities of the RecyclerView for Kodular.

Events

OnWidgetClicked

Event generated when any widget is clicked.For this event to be fired by a widget, it must have the respective property,"Clickable".

NameType(s)Def. value
positionNumber-

groupGroup-

OnBindingCustomView

Event fired when the custom widget that is optionally configured in the RecyclerView is visible on the screen or needs to be updated.

Methods

AddSelectionIconOption

Adds options with icons to the multi-selection bar. Info:

These options will be added to the selection bar. The icon is the name of a file in assets or the path to an image file.

NameType(s)Def. value
titleText-
iconText-

AddSelectionOption

Adds options to the multi-selection bar. Info:

An option to select all items can be set with the SelectAllTitle property.

NameType(s)Def. value
titleText-

Build

Finishes with the recyclerview configuration and adds it inside the specified component. Info:

It is preferable not to add it inside a Scroll Arrangement, but it can be added inside a Swipe Refresh Layout or other arrangement.

NameType(s)Def. value
componentComponent-
dataList-

GetCurrentScrollPixels

Returns the offset position the recyclerview is currently in, the value is returned in pixels.

GetGroupForPosition

Returns the group corresponding to an adapter position (linked to the number of items in the data list). It will only return a valid position if the corresponding group is visible on the screen. Info:

It should be used when we are sure that the item corresponding to the position is visible on the screen, for example after using the ListenableScroll method.

NameType(s)Def. value
positionNumber-

GetLastSyncedWidget

Returns the last widget synced with the RecyclerView. Info:

The returned widget cannot be edited with the Kodular blocks, but it does allow it to be used with extensions such as the Widget Animator or widget Screenshot.

GetRootName

Returns a name that is used to identify the first widget to be added to the RecyclerView model. Info:

This block tells the RecyclerView that it is starting to create the model.

GetScrollRangePixels

Returns the maximum scroll position of the RecyclerView. It is returned in pixels.

ListenableScrollTo

Causes the RecyclerView to go to a certain position and calls the FastScroll event when it is done scrolling.

NameType(s)Def. value
positionNumber-

NotifyDataSetChanged

Notifies the recyclerview that a change occurred in the data list, because of this, it will have to update the widgets.

ScrollTo

Causes the RecyclerView to scroll to the specified position. It does not notify when it is done.

NameType(s)Def. value
positionNumber-

NameType(s)Def. value

SetGridManager

Sets the layout manager to Grid, so that the items in the RecyclerView will be displayed with columns or rows, depending on the orientation.

NameType(s)Def. value

Sets the layout manager to List, so the items in the RecyclerView will be listed horizontally or vertically depending on the orientation.

SetScrollListener

It establishes the procedure that is responsible for listening to the changes of state when scrolling in the RecyclerView. Info:

The addresses are automatically detected depending on the layout manager. In a horizontal manager the procedure should have the following parameters: [canScrollToLeft, canScrollToRight, isIddle] In a vertical manager they should be: [canScrollToTop, canScrollToBottom, isIddle] The isIddle parameter refers to whether the RecyclerView is idle.

NameType(s)Def. value
procedureText-

SetStaggeredGridManager

Sets the layout manager as a staggered grid. It will be displayed in columns or rows depending on the orientation.

NameType(s)Def. value
spanCountNumber-
reverseBoolean-
portraitBoolean-

SetSwipeForViewTypes

It establishes in which direction or in which directions each type of view can be scrolled. Info:

The letters l, r, and b are used to indicate the left, right, and both sides respectively. Rules structure: [view-type]:[side] Example:"4: l, 5: r, 6: b"

NameType(s)Def. value
typesText-

SetViewTypeProvider

It establishes the procedure that is in charge of returning the type of view according to the position that the RecyclerView returns. Info:

The procedure must have the parameters [position, listItem] It must return a type of view, it must be one created previously.

NameType(s)Def. value
procedureText-

SmoothScrollTo

Causes the RecyclerView to scroll smoothly to a specific position. Info:

It does not notify any event if it finished with its task.

NameType(s)Def. value
positionNumber-

Sync

It is mainly used when configuring the RecyclerView for the first time, it tells all the components that depend on it that this RecyclerView is currently being worked on.

The fraction that the user should move the group to be considered as swiped. Block Types: Write Presentation: Designer, Blocks Info:

Values range from 0.0 to 1.0 The fraction is calculated with respect to RecyclerView's bounds.

Default value: 0.5

SwipeVelocityThreshold

Defines the maximum velocity ItemTouchHelper will ever calculate for pointer movements. Block Types: Write Presentation: Designer, Blocks Info:

Values range from 0.0 to 1.0. If you increase the value, it will be easier for the user to swipe diagonally and if you decrease the value, user will need to make a rather straight finger movement to trigger a swipe.

Default value: 1.0
